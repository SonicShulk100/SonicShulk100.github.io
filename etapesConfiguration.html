1) Les prérequis

Si vous voulez créer un projet Symfony, vous devez avoir :

  • Un serveur web (Apache, Tomcat, WAMP (Windows Apache MySQL PHP), etc) : C’est là que fonctionnera votre application construite dans Symfony. Je vous conseille d'installer un WAMP car c'est là où vous écrivez du code en PHP dans le "localhost".
  • Un moteur de bases de données (MySQL, PostgreSQL, SQLite ou tout autre PDO compatible aux bases de données) : Pourquoi avoir une base de données est-elle importante pour votre projet Symfony? C'est parce qu'un projet Symfony se base dans un modèle nommé "MVC" (voir "Introduction à Symfony" pour les détails).
  • Un IDE ("Integrated Development Environment") : C'est là où vous ecrivez du code PHP. Les IDEs les plus connus sont:
    • Visual Studio Code créé par Microsoft
    • PhpStorm créé par JetBrains (PAYANT!!!)
  • PHP 5.2.4 ou plus : Symfony est un framework qui n'est disponible que sur la version 5.2.4 ou plus. Vous pouvez le téléchargez en cliquant sur ce lien.
  • Le CLI ("Command Line Interface") de Symfony : Optionnel mais très util car il peut vous donner les outils nécessaires pour développer et démarrer les applications en local (Compatible uniquement en version 8.2 ou plus).
  • Gestionnaire de dépendances libre « Composer » : Ce gestionnaire permet d'installer les packets PHP qui sont nécéssaires pour votre application web.

2) Etapes d'installation

  1. Installer un serveur web.
  2. Installer un moteur de base de données.
  3. Installer le langage PHP.
  4. Installer l’IDE de votre choix
  5. Installer le CLI de Symfony et Composer

3) Etapes de Configuration

  1. Après avoir installer les prérequis, configurez l'accès au PHP dans votre IDE
  2. Pour créer un projet, allez dans le dossier "www" de votre WAMP et tapez la commande: 
    composer create-project symfony/skeleton:"7.1.*" my_project_directory
  3. Pour installer les packets, vous devez tapez la commande (Dans le dossier du projet) : 
    composer require logger
    (Ceci n'est qu'un exemple. Il existe plusieurs packets que vous pouvez utiliser)
  4. Après, vous pouvez commencer à coder votre propre application web
