Les bonnes bases pour comprendre les bonnes pratiques Les bonnes pratiques en question

1) Les bonnes bases pour comprendre les bonnes pratiques

Il faut d’abord penser que les bonnes pratiques, ça ne veulent pas dire que c’est un ensemble de tutoriels. C’est juste une liste à valider pour que vous puissiez déployer votre application sans crainte. Elle a aussi pour but de vous conseiller et guider, en tant que développeur de manière à uniformiser votre code.

2) Les bonnes pratiques en question

  1. Organiser votre application : Pour être sûr que vous ne perdez pas votre temps à coder, pensez à mettre quels fichiers dans quel dossier. Vous pouvez prendre l’exemple du II.4 dans l’architecture d’un projet Symfony.
  2. Les contrôleurs doivent respecter la règle de « 5-10-20 » :
    • 5 Variables par méthode
    • 10 méthodes maximum par contrôleur
    • 20 lignes de codes par méthodes
    Si vous ne respectez pas cette règle, vous aurez du mal à corriger les bugs qui peuvent exister.
  3. Assurez-vous de faire des tests unitaires ! Ils peuvent être utiles en cas de pannes ou de failles de sécurité. Les outils que vous pouvez utiliser pour les tests sont :
    • PHPUnit et PhpSpec pour le PHP
    • Mink (Qui est une librairie de PHPUnit) et CasperJS pour le JavaScript
